# Social Examples

Reference images for Enablement's LinkedIn visual work. Two subfolders:

| Folder | Purpose | Use as visual direction? |
|---|---|---|
| [`inspiration/`](inspiration/) | Post-redesign production references (3 files: 2 cheat sheets + 1 animated flywheel GIF). The calibration set for `DESIGN_SOCIAL.md`. | **Yes - canonical.** |
| [`archive/`](archive/) | Pre-redesign artefacts (the 4 LinkedIn banners currently on Lanny's profile). Retained for historical reference and for the eventual banner-rebuild brief. | **No - do not calibrate against these.** |

## If you're new here

1. Read [`inspiration/README.md`](inspiration/README.md). It annotates each reference image with what works, what could be improved, and what design rules each one teaches.
2. Then read [`../DESIGN_SOCIAL.md`](../DESIGN_SOCIAL.md). Every rule in it traces back to a reference in `inspiration/`.

## Adding new examples

- **A new post-redesign reference** (e.g. a new cheat sheet that should drive the design system): add to `inspiration/` and annotate it in `inspiration/README.md` using the existing structure (what works / what could be improved / what rules this teaches).
- **A new pre-redesign artefact** (rare - only if you're documenting something legacy): add to `archive/` and note it in `archive/README.md`.
- **Anything that doesn't fit either** (e.g. competitor inspiration, brand mood boards): add a new subfolder with a clear name and a README explaining what's in it.
